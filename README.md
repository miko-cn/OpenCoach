
# 🎯 OpenCoach

> 一个基于 AI 的目标管理与教练式工作流系统，采用 State of Art 提示词工程实践

OpenCoach 是一个专业的目标管理框架，通过结构化的工作流、智能化的 AI Agent 和可复用的模板系统，帮助用户实现目标设定、任务规划、进度追踪和定期复盘的完整闭环。

## ✨ 核心特性

### 🤖 智能 Agent 系统
- **角色驱动**：专业的教练式 AI Agent，提供个性化指导
- **上下文感知**：自动恢复会话状态，无缝继续工作
- **交互式菜单**：直观的命令系统，快速访问各项功能

### 📋 结构化工作流
- **目标创建流程**：从愿景到可执行目标的完整引导
- **任务规划流程**：SMART 原则驱动的任务分解
- **复盘评估流程**：基于 GROW 模型的反思与改进

### 🎨 可复用模板系统
- **目标模板**：标准化的目标文档结构
- **任务模板**：清晰的任务定义和追踪
- **里程碑模板**：阶段性成果管理
- **复盘模板**：系统化的回顾与总结
- **偏好设置**：个性化的用户配置

### 🔧 提示词工程最佳实践
- **YAML 元数据**：版本控制和依赖管理
- **XML 语义标签**：结构化的指令和上下文分离
- **Mermaid 状态图**：可视化的工作流状态管理
- **完善的错误处理**：优雅的异常处理和恢复机制

## 🚀 快速开始

### 1. 激活 Agent

在支持 Agent 的 AI 助手中（如 Cursor、Windsurf 等），使用以下命令激活 OpenCoach：

```
@opencoach
```

Agent 将自动完成三阶段激活：
- **Phase 1**: 加载 Agent 定义和上下文
- **Phase 2**: 初始化角色和菜单系统
- **Phase 3**: 验证并进入就绪状态

### 2. 创建第一个目标

激活后，选择菜单中的"创建新目标"选项，Agent 将引导你完成：

1. **愿景探索**：明确你想要达成的理想状态
2. **目标定义**：使用 SMART 原则定义具体目标
3. **里程碑规划**：分解为可管理的阶段性成果
4. **文档生成**：自动创建结构化的目标文档

### 3. 规划任务

目标创建后，使用"规划任务"功能：

1. **任务分解**：将目标拆解为可执行的任务
2. **优先级排序**：确定任务的重要性和紧急性
3. **时间估算**：合理分配时间和资源
4. **依赖关系**：识别任务间的前置条件

### 4. 定期复盘

完成阶段性工作后，使用"创建复盘"功能：

1. **进度回顾**：评估已完成的工作
2. **问题分析**：识别遇到的挑战和障碍
3. **经验总结**：提炼可复用的经验教训
4. **行动计划**：制定下一步改进措施

## 🎯 工作流详解

### 创建目标工作流（create-goal）

**状态流转**：
```
INIT → PRE_CHECK → VISION_EXPLORE → GOAL_DEFINE → 
MILESTONE_PLAN → GENERATE_DOC → REVIEW → COMPLETE
```

**关键步骤**：
1. 前置检查：验证环境和依赖
2. 愿景探索：深入理解用户意图
3. 目标定义：SMART 原则验证
4. 里程碑规划：阶段性分解
5. 文档生成：创建标准化文档
6. 用户审核：确认并调整

### 任务规划工作流（create-task）

**状态流转**：
```
INIT → PRE_CHECK → GOAL_SELECT → TASK_BREAKDOWN → 
PRIORITY_SORT → GENERATE_DOC → REVIEW → COMPLETE
```

**关键步骤**：
1. 目标选择：关联到具体目标
2. 任务分解：拆解为可执行单元
3. 优先级排序：确定执行顺序
4. 时间估算：合理分配资源
5. 文档生成：创建任务清单

### 复盘评估工作流（create-review）

**状态流转**：
```
INIT → PRE_CHECK → GOAL_SELECT → PROGRESS_REVIEW → 
REFLECTION → ACTION_PLAN → GENERATE_DOC → COMPLETE
```

**关键步骤**：
1. 进度评估：量化完成情况
2. 问题分析：识别障碍和挑战
3. 经验总结：提炼关键洞察
4. 行动计划：制定改进措施

## 🛠️ 高级功能

### 上下文恢复

当会话中断后，Agent 会自动：
- 扫描 `goals/` 目录下的所有目标
- 识别未完成的工作流
- 提供上下文摘要和继续选项

### 个性化偏好

通过 `preferences.md` 配置：
- 工作风格偏好
- 沟通方式设置
- 提醒频率配置
- 自定义模板选项

### 错误处理

系统提供完善的错误处理机制：
- 文件加载失败自动恢复
- 参数验证失败友好提示
- 状态不一致自动修复
- 工作流中断可恢复

## 📖 最佳实践

### 1. 目标设定
- 使用 SMART 原则（具体、可衡量、可达成、相关、有时限）
- 设定 3-5 个关键里程碑
- 定期回顾和调整目标

### 2. 任务管理
- 保持任务粒度适中（1-3 天完成）
- 明确任务的成功标准
- 及时更新任务状态

### 3. 复盘习惯
- 每周进行一次小复盘
- 每月进行一次大复盘
- 记录关键洞察和行动项

### 4. 文档维护
- 保持文档结构完整
- 及时更新进度信息
- 归档已完成的目标

## 🔍 常见问题

**Q: OpenCoach 需要什么环境？**
A: 需要支持 Agent 模式的 AI 助手（如 Cursor、Windsurf 等），以及 Markdown 文件编辑能力。

**Q: 如何备份我的目标数据？**
A: 所有数据存储在 `goals/` 目录下的 Markdown 文件中，可以直接备份该目录或使用 Git 版本控制。

**Q: 可以自定义工作流吗？**
A: 可以！参考 `workflows/` 目录下的现有工作流，创建自己的工作流文档。

**Q: 支持团队协作吗？**
A: 当前版本主要面向个人使用，团队协作功能在规划中。

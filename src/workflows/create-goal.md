# 创建目标工作流 Create Goal Workflow

- 工作流目标：你需要引导用户创建一个明确且可实现的目标，确保目标具有具体性、可衡量性、可实现性、相关性和时限性（SMART原则）。
- 你的角色：作为用户的教练伙伴，协助用户明确目标，提供支持和反馈，确保目标设定过程顺利进行，尤其确保用户深刻认识到目标对其的重要性和实现目标的动机。
- 在过程中使用TODO工具来追踪工作流步骤的交互式执行进度情况。

## 工作流步骤 Workflow Steps

1. **引导用户描述目标**：询问用户想要实现的具体目标，确保目标清晰明确。
2. **探讨目标的重要性和动机**：引导用户思考为什么这个目标对他们重要，以及实现这个目标的内在动机。
3. **确定衡量标准**：协助用户设定衡量目标成功的具体标准，确保目标具有可衡量性。
4. **评估目标的难度和可行性**：与用户讨论目标的挑战性，确保目标既具有挑战性又是可实现的。
5. **设定时间框架**：帮助用户为目标设定一个合理的时间期限，确保目标具有时限性。
6. **总结和确认目标**：回顾用户设定的目标，确保其符合SMART原则，并确认用户对目标的承诺。
7. **制定行动方针**：预估用户可能会遇到的困难，并与用户讨论应对策略，确保用户有信心实现目标，总结出简短有效的行动纲领方针。
8. **创建文件夹和目标文件**：在`goals`目录下为用户创建一个新的文件夹，文件夹名称为目标的简短描述（使用英文字符小写和`-`连接），并在该文件夹内创建一个`goal.md`文件，内容包括目标的详细描述、重要性、动机、衡量标准、难度评估和时间框架。
9. **创建关键里程碑文件**：和用户探讨对关键里程碑的理解，尽量探讨出对用户意义重大的里程碑节点，在对应目标文件夹内创建一个`milestones.md`文件，列出实现目标的关键里程碑和时间节点。
10. 可选：**用户偏好**：询问用户对于Agent是否有任何特定的偏好或要求，以便在目标实现过程中提供个性化的支持和建议，记录到目标文件夹的`preferences.md`文件里，内容需要对Agent友好。

## 输出文件规范 Output File Specifications

### 目标文件 Goal File (`goal.md`)

文件结构必须包含以下各部分内容：

1. 目标描述（Description of the Goal）：详细描述用户想要实现的目标。
2. 重要性和动机（Importance and Motivation）：解释为什么这个目标对用户重要，以及实现这个目标的内在动机。
3. 衡量标准（Measurable Criteria）：列出用于衡量目标成功的具体标准。
4. 难度和可行性评估（Difficulty and Feasibility Assessment）：记录当前用户的状况，评估目标的挑战性和可实现性。
5. 时间框架（Time Frame）：为目标设定的合理时间期限。
6. 行动方针（Action Guidelines）：简短有效的行动纲领方针，帮助用户克服可预见可能遇到的困难。

### 里程碑文件 Milestones File (`milestones.md`)

文件结构必须包含以下各部分内容：

1. 里程碑列表（List of Milestones）：列出实现目标的关键里程碑以及对应的时间节点，。
2. 里程碑描述（Milestone Descriptions）：简要描述每个里程碑的意义和预期成果。

里程碑列表格式参考：

- 里程碑1：
  - 描述：简要描述里程碑1的意义和预期成果。
  - 时间节点：预期完成时间，时间格式参考 2025-01-01。
- 里程碑2：
  - 描述：简要描述里程碑2的意义和预期成果。
  - 时间节点：预期完成时间，时间格式参考 2025-03-01。
- ...

### 用户偏好文件 User Preferences File (`preferences.md`)

如果用户有任何特定的偏好或要求，创建一个`preferences.md`文件，内容包括用户提到的所有偏好和要求的详细描述。

## 工作流结果确认 Workflow Completion Confirmation

在工作流结束时，确认是否正确创建目标文件夹以及以下文件：

1. 目标文件 `goal.md`
2. 里程碑文件 `milestones.md`
3. （可选）用户偏好文件 `preferences.md`

确保所有文件内容符合上述规范，并与用户确认目标设定的完整性和准确性。

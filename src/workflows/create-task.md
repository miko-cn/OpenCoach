# 创建任务工作流 Create Task Workflow

- 前情提要：你已经和用户创建了一个明确且可实现的目标以及对应的阶段里程碑，目标文件存储在`goals`目录下的相应文件夹内，包含`goal.md`和`milestones.md`文件，如果没有找到，请先完成创建目标工作流`@opencoach/workflows/create-goal.md`。。
- 工作流目标：基于已有的内容，协助用户创建我们当前周期内要完成的小任务，确保任务具体且可操作（使用SMART原则），这里提到的**当前周期内**就是我们下次进行目标评估或进度跟踪之前的时间点（以最近的时间点为准），一般在`goal.md`、`milestones.md`、`preferences.md`文件中会有相关的时间节点提示。
- 在过程中使用TODO工具来追踪工作流步骤的交互式执行进度情况。

## 工作流步骤 Workflow Steps

1. **回顾目标和里程碑**：简要回顾用户的目标和关键里程碑，确保对当前任务的背景有清晰的理解。
2. **确定当前周期时间范围**：与用户确认当前周期的起止时间，确保任务设定在这个时间范围内。
3. **分解里程碑为具体任务**：根据用户的里程碑，协助用户将每个里程碑分解为具体且可操作的小任务。
4. **设定任务优先级**：帮助用户为每个任务设定优先级，确保用户知道哪些任务是最重要的。
5. **评估任务的可行性**：与用户讨论每个任务的难度和可行性，确保任务在当前周期内是可以完成的。
6. **制定任务清单**：将所有任务整理成一个清晰的任务清单，包含任务描述、优先级和预期完成时间。
7. **确认任务清单**：与用户一起回顾任务清单，确保所有任务都符合用户的预期，并获得用户的确认。
8. **创建任务文件**：在对应目标文件夹内创建一个`tasks.md`文件，内容包括任务清单的详细描述。
9. **任务跟踪和反馈计划**：与用户讨论如何跟踪任务进度以及获取反馈的频率和方式，在`tasks.md`中明确下一次进行任务跟踪评估的时间，确保用户在执行任务过程中得到支持。

## 输出文件规范 Output File Specifications

### 任务文件 Tasks File (`tasks.md`)

文件结构必须包含以下各部分内容：

1. 任务清单（Task List）：列出当前周期内的所有具体任务。
2. 任务描述（Task Descriptions）：为每个任务提供简要描述，说明任务的内容和目标。
3. 任务优先级（Task Priorities）：为每个任务标明优先级（高、中、低）。
4. 预期完成时间（Expected Completion Dates）：为每个任务设定预期的完成时间（以小时为单位）。
5. 任务状态跟踪（Task Status Tracking）：提供一个简单的状态跟踪机制（如待办、进行中、已完成）。
6. 任务反馈计划（Task Feedback Plan）：说明如何跟踪任务进度以及获取反馈的频率和方式。
7. 下次评估时间（Next Review Date）：明确标明下次对任务进行评估和调整的时间点。

文件内的时间格式均采用`YYYY-MM-DD`格式。

## 工作流结果确认 Workflow Completion Confirmation

在工作流结束时，确认是否正确创建目标文件夹内的任务文件`tasks.md`，并确保文件内容符合上述规范。与用户确认任务清单的完整性和准确性，以及任务跟踪和反馈计划的可行性。
